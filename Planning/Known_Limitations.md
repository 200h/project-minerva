# Known Limitations

**Last Updated:** 2026-07-21  
**Owner:** Product Owner and Technical Director

## Current Project Limitations

### No Runtime Implementation

The repository currently contains governing and workflow documentation only. No playable Unity project or runtime systems have been committed.

### No Approved Architecture Baseline

Detailed repository structure, runtime layers, system contracts, and data flows are not yet approved. Implementation tickets must not invent them.

### No Active Codex Implementation Tickets

The ticket workflow exists, but no implementation task should enter Ready until the architecture baseline and required context are approved.

### No Third-Party Assets Approved

Potential assets may be discussed in the Constitution, but none are approved, licensed, installed, or wrapped for production use.

### Unity Compatibility Details Incomplete

Unity 5.6 is authoritative, but exact scripting runtime, platform toolchain, and package compatibility records still require verification.

### No Automated Build or Test Pipeline

CI, automated Unity builds, and test reporting have not been established.

### No Production Content Schema

Story event, condition, effect, branch, merge, character, conversation, and phone-state schemas remain to be designed.

## Maintenance Rule

Remove a limitation only when the replacement capability is merged and validated. Link the resolving PR, ticket, or ADR when practical.
